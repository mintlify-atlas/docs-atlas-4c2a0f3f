## Prerequisites

Before you begin, make sure you have:

- **Docker** 20+ and **Docker Compose** v2 (`docker compose version`)
- **Git** to clone the repository
- A **PostgreSQL** database (or use an existing Azure Flexible Server)
- A **MercadoPago** account (for payment features)
- A **Cloudinary** account (for image uploads)

## 1. Clone the repository

```bash
git clone https://github.com/ubik-back/ubik-backend.git
cd ubik-backend/microservicios/microreactivo
```

## 2. Configure environment variables

Create a `.env` file in the `microservicios/microreactivo/` directory with all required variables:

```bash .env
# Database (R2DBC PostgreSQL)
SPRING_R2DBC_URL=r2dbc:postgresql://localhost:5432/ubikdb
SPRING_R2DBC_USERNAME=postgres
SPRING_R2DBC_PASSWORD=yourpassword

# JWT
JWT_SECRET=your-256-bit-secret-key-here
JWT_EXPIRATION=86400000

# Role IDs (numeric IDs from your roles table)
ROLE_ID_ADMIN=1
ROLE_ID_PROPERTY_OWNER=2
ROLE_ID_USER=3

# Cloudinary
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret

# MercadoPago
MERCADOPAGO_ACCESS_TOKEN=your-access-token
MERCADOPAGO_WEBHOOK_SECRET=your-webhook-secret
MERCADOPAGO_CLIENT_ID=your-client-id
MERCADOPAGO_CLIENT_SECRET=your-client-secret
MERCADOPAGO_REDIRECT_URI=https://yourapp.com/oauth/callback
MP_FEE_PERCENT=10

# Mail (SMTP)
MAIL_USERNAME=your@gmail.com
MAIL_PASSWORD=your-app-password
```

<Note>
  `JWT_EXPIRATION` is in milliseconds. The default `86400000` equals 24 hours.
</Note>

## 3. Initialize the database schema

Run the SQL initialization script against your PostgreSQL instance before starting the services:

```bash
psql -U postgres -d ubikdb -f motelManegement/src/main/resources/Postgres-init-motel.sql
```

## 4. Start all services

```bash
docker compose up --build
```

Docker Compose starts the services in dependency order:

1. `usermanagement-service` (port 8081)
2. `motel-management-service` (port 8083)
3. `notification-service` (port 8084)
4. `payment-service` (port 8085)
5. `gateway` (port 8080) — starts last, after all services are healthy

Wait for all health checks to pass. You can monitor progress with:

```bash
docker compose logs -f gateway
```

The gateway is ready when you see it pass its health check at `http://localhost:8080/actuator/health`.

## 5. Register your first user

All requests go through the gateway on port **8080**.

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john",
    "email": "john@example.com",
    "password": "secret123",
    "phoneNumber": "+1234567890",
    "anonymous": false,
    "roleId": 3,
    "birthDate": "1990-01-15"
  }'
```

A successful response returns a JWT token string.

## 6. Log in and get your token

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "john", "password": "secret123"}'
```

Save the returned JWT token:

```bash
TOKEN="eyJhbGciOiJIUzI1NiJ9..."
```

## 7. Make your first authenticated request

Fetch your user profile:

```bash
curl http://localhost:8080/api/user \
  -H "Authorization: Bearer $TOKEN"
```

## Service ports

| Service | Port | Purpose |
|---|---|---|
| Gateway | 8080 | Single entry point for all API requests |
| User Management | 8081 | Auth, registration, profiles |
| Motel Management | 8083 | Motels, rooms, reservations, services |
| Notification | 8084 | Email delivery |
| Payment | 8085 | MercadoPago integration |
