## Overview

Three public endpoints for listing rooms. No authentication is required for any of them.

- `GET /api/rooms` — returns every room in the system.
- `GET /api/rooms/motel/{motelId}` — returns all rooms belonging to a specific motel.
- `GET /api/rooms/motel/{motelId}/available` — returns only rooms that are currently available for the given motel.

---

## GET /api/rooms

Returns all rooms across all motels.

**Method:** `GET`  
**Path:** `/api/rooms`  
**Auth required:** No

### Example request

```bash
curl -X GET http://localhost:8080/api/rooms
```

### Response — `200 OK`

Returns a JSON array of `RoomResponse` objects. See [Response body fields](#response-body-fields) below.

---

## GET /api/rooms/motel/{motelId}

Returns all rooms (available and unavailable) belonging to a specific motel.

**Method:** `GET`  
**Path:** `/api/rooms/motel/{motelId}`  
**Auth required:** No

### Path parameters

<ParamField path="motelId" type="long" required>
  The unique identifier of the motel whose rooms should be listed.
</ParamField>

### Example request

```bash
curl -X GET http://localhost:8080/api/rooms/motel/10
```

---

## GET /api/rooms/motel/{motelId}/available

Returns only rooms with `isAvailable: true` for the specified motel.

**Method:** `GET`  
**Path:** `/api/rooms/motel/{motelId}/available`  
**Auth required:** No

### Path parameters

<ParamField path="motelId" type="long" required>
  The unique identifier of the motel whose available rooms should be listed.
</ParamField>

### Example request

```bash
curl -X GET http://localhost:8080/api/rooms/motel/10/available
```

---

## Response body fields

All three endpoints return arrays of the same `RoomResponse` object:

<ResponseField name="id" type="long">
  Unique identifier of the room.
</ResponseField>

<ResponseField name="motelId" type="long">
  ID of the motel this room belongs to.
</ResponseField>

<ResponseField name="number" type="string">
  Room number or identifier within the motel (e.g., `"101"`, `"Suite A"`).
</ResponseField>

<ResponseField name="roomType" type="string">
  Type/category of the room (e.g., `"Standard"`, `"Suite"`, `"Deluxe"`).
</ResponseField>

<ResponseField name="price" type="double">
  Price per unit (see motel pricing configuration for the unit — hour or night).
</ResponseField>

<ResponseField name="description" type="string">
  Optional description of the room.
</ResponseField>

<ResponseField name="isAvailable" type="boolean">
  Whether the room is currently available for booking.
</ResponseField>

<ResponseField name="imageUrls" type="array of string">
  List of image URLs for the room. Maximum 15 entries.
</ResponseField>

<ResponseField name="latitude" type="double">
  Geographic latitude of the room/property location.
</ResponseField>

<ResponseField name="longitude" type="double">
  Geographic longitude of the room/property location.
</ResponseField>

<ResponseField name="motelName" type="string">
  Name of the parent motel, denormalized for convenience.
</ResponseField>

<ResponseField name="motelAddress" type="string">
  Street address of the parent motel.
</ResponseField>

<ResponseField name="motelCity" type="string">
  City of the parent motel.
</ResponseField>

<ResponseField name="motelPhoneNumber" type="string">
  Contact phone number of the parent motel.
</ResponseField>

<ResponseField name="serviceIds" type="array of long">
  IDs of services associated with this room.
</ResponseField>

## Error responses

| Status | Description |
|--------|-------------|
| 404 | No motel found with the given `motelId` (for motel-scoped endpoints) |
