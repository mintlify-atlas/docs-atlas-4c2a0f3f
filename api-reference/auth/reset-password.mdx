---
title: "Password Reset"
description: "Request a password reset email and complete the reset using a token."
---

## Overview

Password reset is a two-step flow:

1. **Request a reset** — submit the account email address. The server sends a password reset email containing a time-limited token.
2. **Confirm the reset** — submit the token from the email together with the new password.

---

## Step 1 — Request password reset

**Method:** POST  
**Path:** `/api/auth/reset-password-request`  
**Auth required:** No

### Query parameters

<ParamField query="email" type="string" required>
  The email address associated with the account.
</ParamField>

### Example request

```bash
curl -X POST "http://localhost:8080/api/auth/reset-password-request?email=john@example.com"
```

### Response

**Status:** 200 OK

Returns a confirmation message as a plain string.

```json
"Password reset email sent"
```

<Info>
  The reset token included in the email has a limited validity window. Request a new one if it expires before use.
</Info>

---

## Step 2 — Confirm password reset

**Method:** POST  
**Path:** `/api/auth/reset-password`  
**Auth required:** No

### Request body

<ParamField body="token" type="string" required>
  The password reset token received by email.
</ParamField>

<ParamField body="newPassword" type="string" required>
  The new password to set for the account.
</ParamField>

### Example request

```bash
curl -X POST http://localhost:8080/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "abc123resettoken",
    "newPassword": "newSecurePass1"
  }'
```

### Response

**Status:** 200 OK

Returns a success message as a plain string.

```json
"Password reset successfully"
```

## Error responses

| Status | Description |
|--------|-------------|
| 400 | Validation error — `token` or `newPassword` is blank, or the token is invalid/expired |
