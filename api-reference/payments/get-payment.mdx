## Overview

This page documents three endpoints for retrieving payment records: by internal payment ID, by reservation ID, and by user ID.

---

## GET /api/payments/{id}

Returns a single payment record by its internal ID.

**Method:** GET  
**Path:** `/api/payments/{id}`  
**Auth required:** Yes  

### Path parameters

<ParamField path="id" type="long" required>
  The internal payment ID.
</ParamField>

### Example request

```bash
curl http://localhost:8080/api/payments/9 \
  -H "Authorization: Bearer <token>"
```

### Example response

**Status:** 200 OK

```json
{
  "id": 9,
  "reservationId": 47,
  "userId": 3,
  "amount": 250000.0,
  "currency": "COP",
  "status": "APPROVED",
  "initPoint": "https://www.mercadopago.com.co/checkout/v1/redirect?pref_id=12345678-abcd-...",
  "mercadoPagoPaymentId": "7891234560",
  "failureReason": null,
  "createdAt": "2026-03-17T10:45:00",
  "updatedAt": "2026-03-17T10:52:00"
}
```

---

## GET /api/payments/reservation/{reservationId}

Returns all payment records associated with a specific reservation. A reservation may have more than one payment record if previous attempts were rejected.

**Method:** GET  
**Path:** `/api/payments/reservation/{reservationId}`  
**Auth required:** Yes  

### Path parameters

<ParamField path="reservationId" type="long" required>
  The ID of the reservation whose payments to retrieve.
</ParamField>

### Example request

```bash
curl http://localhost:8080/api/payments/reservation/47 \
  -H "Authorization: Bearer <token>"
```

### Example response

**Status:** 200 OK — returns a JSON array

```json
[
  {
    "id": 9,
    "reservationId": 47,
    "userId": 3,
    "amount": 250000.0,
    "currency": "COP",
    "status": "APPROVED",
    "initPoint": "https://www.mercadopago.com.co/checkout/v1/redirect?pref_id=12345678-abcd-...",
    "mercadoPagoPaymentId": "7891234560",
    "failureReason": null,
    "createdAt": "2026-03-17T10:45:00",
    "updatedAt": "2026-03-17T10:52:00"
  }
]
```

## GET /api/payments/user/{userId}

Returns all payment records for a specific user across all their reservations.

**Method:** GET  
**Path:** `/api/payments/user/{userId}`  
**Auth required:** Yes  

### Path parameters

<ParamField path="userId" type="long" required>
  The internal user ID whose payments to retrieve.
</ParamField>

### Example request

```bash
curl http://localhost:8080/api/payments/user/3 \
  -H "Authorization: Bearer <token>"
```

### Example response

**Status:** 200 OK — returns a JSON array

```json
[
  {
    "id": 9,
    "reservationId": 47,
    "userId": 3,
    "amount": 250000.0,
    "currency": "COP",
    "status": "APPROVED",
    "initPoint": "https://www.mercadopago.com.co/checkout/v1/redirect?pref_id=12345678-abcd-...",
    "mercadoPagoPaymentId": "7891234560",
    "failureReason": null,
    "createdAt": "2026-03-17T10:45:00",
    "updatedAt": "2026-03-17T10:52:00"
  }
]
```

## Response fields

<ResponseField name="id" type="long">
  Internal payment ID.
</ResponseField>

<ResponseField name="reservationId" type="long">
  The reservation this payment is associated with.
</ResponseField>

<ResponseField name="userId" type="long">
  The user who made the payment.
</ResponseField>

<ResponseField name="amount" type="double">
  The payment amount.
</ResponseField>

<ResponseField name="currency" type="string">
  The currency code (e.g. `"COP"`).
</ResponseField>

<ResponseField name="status" type="string">
  Current payment status: `PENDING`, `APPROVED`, `REJECTED`, `REFUNDED`, or `CANCELLED`.
</ResponseField>

<ResponseField name="initPoint" type="string">
  The MercadoPago checkout URL originally generated for this payment.
</ResponseField>

<ResponseField name="mercadoPagoPaymentId" type="string">
  The MercadoPago payment ID assigned after the user completes checkout. `null` if payment has not been processed yet.
</ResponseField>

<ResponseField name="failureReason" type="string">
  Reason for rejection provided by MercadoPago. `null` if the payment was not rejected.
</ResponseField>

<ResponseField name="createdAt" type="string (ISO 8601 datetime)">
  Timestamp of payment creation.
</ResponseField>

<ResponseField name="updatedAt" type="string (ISO 8601 datetime)">
  Timestamp of last status update.
</ResponseField>

## Error responses

| Status | Description |
|--------|-------------|
| 401 | Missing or invalid authentication token |
| 404 | Payment not found for the given ID |
