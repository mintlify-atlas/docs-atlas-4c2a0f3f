---
title: "GET /api/admin/motels/pending"
description: "Retrieve all motels currently in PENDING or UNDER_REVIEW approval status."
---

## Overview

Returns the full list of motels that are awaiting an admin decision — those with an approval status of `PENDING` or `UNDER_REVIEW`. Use this endpoint as the starting point for the admin moderation queue.

<Note>
  This endpoint requires the `ADMIN` role. Requests without a valid admin role header are rejected with `403 Forbidden`.
</Note>

## Request

**Method:** `GET`  
**Path:** `/api/admin/motels/pending`  
**Auth required:** Yes  
**Required role:** `ADMIN`

### Required headers

<ParamField header="X-User-Role" type="string" required>
  The role ID for the authenticated user. Must match the configured admin role ID.
</ParamField>

<ParamField header="X-User-Username" type="string" required>
  The username of the authenticated admin user, forwarded by the API gateway.
</ParamField>

<ParamField header="X-User-Id" type="string">
  The numeric user ID of the authenticated admin, forwarded by the API gateway.
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token issued by the authentication service.
</ParamField>

### Example request

```bash
curl -X GET http://localhost:8080/api/admin/motels/pending \
  -H "Authorization: Bearer <token>" \
  -H "X-User-Role: <admin-role-id>" \
  -H "X-User-Username: admin_user" \
  -H "X-User-Id: 1"
```

## Response

**Status:** `200 OK`  
Returns a JSON array of `MotelResponse` objects. The array is empty when no motels are pending.

### Response body fields

Each element in the array is a motel object:

<ResponseField name="id" type="long">
  Unique identifier of the motel.
</ResponseField>

<ResponseField name="name" type="string">
  Name of the motel.
</ResponseField>

<ResponseField name="approvalStatus" type="string">
  Current approval status. For this endpoint, always `PENDING` or `UNDER_REVIEW`.
</ResponseField>

<ResponseField name="ownerId" type="long">
  ID of the property owner who registered the motel.
</ResponseField>

<ResponseField name="address" type="string">
  Street address of the motel.
</ResponseField>

<ResponseField name="city" type="string">
  City where the motel is located.
</ResponseField>

<ResponseField name="phoneNumber" type="string">
  Contact phone number for the motel.
</ResponseField>

## Error responses

| Status | Description |
|--------|-------------|
| 401 | Missing `X-User-Role` or `X-User-Username` header — user is not authenticated |
| 403 | Authenticated user does not have the admin role |
| 400 | `X-User-Id` header is present but not a valid number |
