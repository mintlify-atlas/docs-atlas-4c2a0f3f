## Overview

Four public read endpoints for services. No authentication is required.

- `GET /api/services` вЂ” returns all services.
- `GET /api/services/{id}` вЂ” returns a single service by ID.
- `GET /api/services/name/{name}` вЂ” returns a single service by exact name.
- `GET /api/services/room/{roomId}` вЂ” returns the IDs of all services associated with a room.

---

## GET /api/services

Returns all services in the system.

**Method:** `GET`  
**Path:** `/api/services`  
**Auth required:** No

### Example request

```bash
curl -X GET http://localhost:8080/api/services
```

### Response вЂ” `200 OK`

Returns a JSON array of `ServiceResponse` objects. See [Response body fields](#response-body-fields) below.

---

## GET /api/services/{id}

Returns a single service by its unique identifier.

**Method:** `GET`  
**Path:** `/api/services/{id}`  
**Auth required:** No

### Path parameters

<ParamField path="id" type="long" required>
  The unique identifier of the service to retrieve.
</ParamField>

### Example request

```bash
curl -X GET http://localhost:8080/api/services/3
```

---

## GET /api/services/name/{name}

Looks up a service by its exact name. The match is performed by the service layer and is typically case-sensitive.

**Method:** `GET`  
**Path:** `/api/services/name/{name}`  
**Auth required:** No

### Path parameters

<ParamField path="name" type="string" required>
  The exact name of the service to find (e.g., `WiFi`, `Jacuzzi`, `Desayuno`).
</ParamField>

### Example request

```bash
curl -X GET http://localhost:8080/api/services/name/WiFi
```

---

## GET /api/services/room/{roomId}

Returns a list of service **IDs** (not full service objects) associated with a given room. Use `GET /api/services/{id}` to fetch details for each ID.

**Method:** `GET`  
**Path:** `/api/services/room/{roomId}`  
**Auth required:** No

### Path parameters

<ParamField path="roomId" type="long" required>
  The unique identifier of the room whose service IDs should be returned.
</ParamField>

### Example request

```bash
curl -X GET http://localhost:8080/api/services/room/5
```

### Response вЂ” `200 OK`

Returns a JSON array of `long` values:

```json
[1, 3, 7]
```

---

## Response body fields

For `GET /api/services`, `GET /api/services/{id}`, and `GET /api/services/name/{name}`:

<ResponseField name="id" type="long">
  Unique identifier of the service.
</ResponseField>

<ResponseField name="name" type="string">
  Name of the service (e.g., `"WiFi"`, `"Jacuzzi"`, `"Desayuno incluido"`).
</ResponseField>

<ResponseField name="description" type="string">
  Optional description of the service. `null` if not set.
</ResponseField>

<ResponseField name="icon" type="string">
  Optional icon identifier or name for UI display (e.g., `"wifi"`, `"hot-tub"`). `null` if not set.
</ResponseField>

<ResponseField name="createdAt" type="string (ISO-8601 datetime)">
  Timestamp of when the service was created.
</ResponseField>

## Error responses

| Status | Description |
|--------|-------------|
| 404 | No service found with the given `id` or `name`; or no room found with the given `roomId` |
